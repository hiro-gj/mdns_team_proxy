# 機能要件一覧

> バージョン: 1 | 更新日時: 2026/6/9 12:38:00

## [FR-001] 排他制御による二重起動防止機能

- **優先度:** 高
- **説明:** Daemonモードでの起動時にロックファイル（mdns_proxy.lock）を用いてプロセスの重複起動を検知し、多重起動を防止する。
**sourceEvidence:**

- mdns_proxy/src/main.py

- **relatedRequirement:** BR-001

## [FR-002] プラットフォーム判定とMicroPython Wi-Fi接続機能

- **優先度:** 中
- **説明:** 起動プラットフォームがPico(MicroPython)かOS(Windows/Linux)かを自動判定し、Pico環境の場合に限り、指定されたSSIDとパスワードを用いてWi-Fiに自動接続を行う。
**sourceEvidence:**

- mdns_proxy/src/main.py
- mdns_proxy/src/wifi_manager.py

- **relatedRequirement:** BR-002

## [FR-003] データベース初期化とWALモード設定機能

- **優先度:** 高
- **説明:** SQLite3データベース（mdns_proxy.sqlite3）を構築し、static_hosts, self_records, other_records, merged_records テーブルを初期化する。また、排他制御や同時アクセス性能を高めるためにジャーナルモードをWAL、同期モードをNORMALに設定する。
**sourceEvidence:**

- mdns_proxy/src/database.py

- **relatedRequirement:** BR-003

## [FR-004] 対話型CLI画面におけるマージ済みレコード表示機能

- **優先度:** 中
- **説明:** 対話型CLI画面から「マージ済みレコードの表示」を選択し、データベース内の merged_records テーブルに格納されたレコードを一覧形式でターミナルに出力する。
**sourceEvidence:**

- mdns_proxy/src/cli.py

- **relatedRequirement:** BR-004

## [FR-005] 対話型CLI画面における静的ホスト管理機能（一覧・追加・削除）

- **優先度:** 中
- **説明:** 対話型CLI画面（静的ホスト一覧画面, 静的ホスト追加画面, 静的ホスト削除画面）を介して、static_hosts エンティティに登録されたホスト情報の閲覧、新規ホストのIPアドレス紐付け（省略時は自動解決）、および登録済みの静的ホストの削除を行う。
**sourceEvidence:**

- mdns_proxy/src/cli.py

- **relatedRequirement:** BR-005

## [FR-006] API /api/merged-records によるマージ済みレコード配信機能

- **優先度:** 高
- **説明:** HTTP GETリクエストを受け取った際、データベースの merged_records からホスト名、IPアドレス、レコードタイプ、TTLの情報を取得し、JSON形式でクライアントに返却する。
**sourceEvidence:**

- mdns_proxy/src/api_server.py

- **relatedRequirement:** BR-006

## [FR-007] API /api/other-records による他プロキシからの同期情報受信機能

- **優先度:** 高
- **説明:** 他のmDNS ProxyからHTTP POSTリクエストを介して送信される、self_recordsおよびother_recordsの同期パケットを受信する。トークン認証および自ノードID検証を行い、受信レコードをother_recordsテーブルに格納する。
**sourceEvidence:**

- mdns_proxy/src/api_server.py

- **relatedRequirement:** BR-007

## [FR-008] 独自DNS名前解決および自己参照ループ防止機能

- **優先度:** 高
- **説明:** static_hostsに登録されているホストに対して、システムDNSキャッシュを介さずに生mDNSクエリまたはpingで名前解決を試み、self_recordsを更新する。この際、ループバックアドレスは除外するとともに、自Proxyが解決した古いレコードの再解決による自己参照ループを回避する。
**sourceEvidence:**

- mdns_proxy/src/dns_resolver.py

- **relatedRequirement:** BR-008

## [FR-009] mDNSマルチキャストクエリ応答機能

- **優先度:** 高
- **説明:** UDP 5353ポートでマルチキャストを待ち受け、mDNSによるホスト名解決クエリに対して、マージ済みレコードや自ノード、静的ホストに基づいたIPアドレスを含む応答をパケットとして返信する。サービスタイプクエリは応答対象外として早期リターンする。
**sourceEvidence:**

- mdns_proxy/src/mdns_server.py

- **relatedRequirement:** BR-009

## [FR-010] スケジューラによる常駐処理（同期・TTL減算・マージ処理）

- **優先度:** 高
- **説明:** デーモンまたは常駐プロセスとして起動し、設定されたインターバル周期で以下のバックグラウンド処理を行う。1. TTL減算とクリーンアップ、2. 静的ホストの独自解決（self_records登録）、3. 固定IPによる他プロキシ発見、4. 中継・収束方式に基づく他ノードへのデータ送信、5. merged_recordsへのマージ処理。
**sourceEvidence:**

- mdns_proxy/src/scheduler.py

- **relatedRequirement:** BR-010

## [FR-011] ログファイルのタイムスタンプローテーションとZIP自動圧縮機能

- **優先度:** 低
- **説明:** 動作ログ（mdns_proxy.log）を出力し、日次などでタイミングローテーションを行う。ローテーションされた過去のログファイルをZIP圧縮（.zip）してアーカイブ化し、保持件数上限を超えた古い圧縮ファイルを自動削除する。
**sourceEvidence:**

- mdns_proxy/src/logger_config.py

- **relatedRequirement:** BR-011
