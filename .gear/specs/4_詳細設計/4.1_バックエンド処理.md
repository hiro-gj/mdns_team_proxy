## [BP-001] システム初期化管理

- **カテゴリ:** マスタ管理
**relatedScreens:**

- FR-001

- **説明:** システム起動時の設定ファイル読み込み、SQLite3データベースの初期化、テーブル作成、初期データセットアップを行う機能
**relatedTables:**

- static_hosts
- self_records
- other_records
- other_proxies
- merged_records
- system_config
- system_logs
- dns_resolution_log

**security:**

- auth: システムレベル
- permissions: system_init

**operations:**

- **システム初期化**
  - type: CREATE
  - トリガー: システム起動時
- **データベーステーブル作成**
  - type: CREATE
  - トリガー: 初期化処理内

- **備考:** システム起動時に1回のみ実行される重要な初期化処理。名前解決は自プロキシ内でdig、Resolve-DnsName、ping、arpingコマンド相当の処理で実行

## [BP-007] DNS名前解決処理

- **カテゴリ:** 外部連携
**relatedScreens:**

- FR-009

- **説明:** static_DBのホスト名について、DNS依存では無くmDNSプロキシ自身でdig、Resolve-DnsName、ping、arpingコマンド相当の正引きを実行し、取得したIPアドレスをself_DBに格納する機能
**relatedTables:**

- static_hosts
- self_records
- dns_resolution_log

**security:**

- auth: システム内部処理
- permissions: dns_resolution

**operations:**

- **独自名前解決実行**
  - type: SEARCH
  - トリガー: 定期実行スケジューラからの呼び出し
- **dig相当処理**
  - type: SEARCH
  - トリガー: 独自名前解決の第一段階
- **Resolve-DnsName相当処理**
  - type: SEARCH
  - トリガー: dig相当処理失敗時のフォールバック
- **ping相当処理**
  - type: SEARCH
  - トリガー: DNS解決失敗時のICMP確認
- **arping相当処理**
  - type: SEARCH
  - トリガー: ローカルネットワーク内のARP解決
- **解決結果記録・更新**
  - type: UPDATE
  - トリガー: 名前解決成功時の内部処理
- **解決失敗処理**
  - type: DELETE
  - トリガー: 全手法による名前解決失敗時

- **備考:** DNS依存を排除した独自実装により、外部DNS障害時でも名前解決を継続。複数手法のフォールバック機能で信頼性を確保
