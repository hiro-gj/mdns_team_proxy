# 業務要件一覧

> バージョン: 2 | 更新日時: 2026/06/11 09:22:00

| id | category | requirement | description | priority |
| --- | --- | --- | --- | --- |
| BR-001 | システム初期化 | 設定ファイル読み込み | システム起動時に検索ホスト名.iniとシステム.iniから設定情報を読み込み、static_DBを初期化する | 高 |
| BR-002 | システム初期化 | データベース初期化 | SQLite3を使用してstatic_DB、self_DB、other_DB、other_proxy_DB、merged_DBを初期化する | 高 |
| BR-003 | プロキシ発見 | 固定IPプロキシ発見 | システム.iniに設定された固定IPリストから外部mDNSプロキシを発見し、other_proxy_DBに登録する | 高 |
| BR-004 | プロキシ発見 | トークンベースプロキシ発見 | 発信トークン（接頭語+MACアドレス）による受動的なプロキシ発見を行い、other_proxy_DBを更新する | 中 |
| BR-005 | データ同期 | static_DB共有 | 発見した全プロキシ間で最新タイムスタンプのstatic_DBを共有・更新する | 高 |
| BR-006 | 名前解決 | self_DB更新 | 最新のstatic_DBを元に名前解決を行い、取得したIP（IPv4/IPv6）をmDNS hosts形式でself_DBに保存・更新する | 高 |
| BR-007 | データ同期 | other_DB取得 | 外部プロキシからself_DB相当データをHTTP（JSON形式）で取得し、other_DBを更新する | 高 |
| BR-008 | データマージ | merged_DB作成 | self_DBとother_DBをマージしてmerged_DBを作成する。重複発生時はタイムスタンプ最新優先でマージする | 高 |
| BR-009 | mDNS応答 | mDNS問い合わせ処理 | UDP/5353でmDNS問い合わせを受信し、merged_DBから該当レコードを検索してmDNS応答を返す。また、Windowsリゾルバ対策として既存ソケットの送信元ポート5353でのユニキャスト応答に加え、マルチキャスト（224.0.0.251:5353）に対しても同時に同一応答をブロードキャストする | 高 |
| BR-010 | 認証 | トークン認証 | プロキシ間通信時にHTTPヘッダで発信トークンを送信し、受信側でトークン認証を行う | 中 |
| BR-011 | データ管理 | TTL管理 | レコードに初期値120秒のTTLを設定し、定期的にTTL切れレコードを削除する | 中 |
| BR-012 | スケジュール実行 | 定期実行ループ | 設定された実行間隔（デフォルト30秒）で一連の処理（プロキシ発見、データ同期、名前解決、mDNS応答）をループ実行する | 高 |
| BR-013 | エラー処理 | 外部プロキシ接続エラー処理 | 外部プロキシからのデータ取得に失敗した場合、該当プロキシをスキップして次サイクルで再試行する | 中 |
| BR-014 | ログ記録 | ローカルログ記録 | システムの動作状況とエラー情報をINFO/ERRORレベルでローカルファイルに記録する | 低 |
| BR-015 | 管理インタフェース | CLI管理機能 | 対話メニュー形式のCLIインタフェースで設定変更や管理操作を提供する | 中 |
| BR-016 | プラットフォーム対応 | Raspberry Pi Pico対応 | MicroPython（Pico版）でWi-Fi接続を使用して動作する | 高 |
| BR-017 | プラットフォーム対応 | Windows EXE対応 | PyInstallerで生成されたWindows用EXEファイルとして動作し、Pico版と設定互換性を保つ | 高 |
| BR-018 | 会議・承認 | 未定義 | ※要確認：業務承認フローや会議体に関する情報が未提供 | 未定義 |
| BR-019 | 運用ルール | 未定義 | ※要確認：システム運用時の業務ルールや制約事項が未提供 | 未定義 |
| BR-020 | ユーザー管理 | 未定義 | ※要確認：管理者権限や利用者管理に関する業務要件が未提供 | 未定義 |
